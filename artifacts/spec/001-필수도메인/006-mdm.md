# 기준 정보(MDM: Master Data Management) 도메인 모델

## 모델링 바이브

- MDM 바이브
    - System Dictionary (시스템 사전): 시스템 내 모든 엔티티가 공유하는 공용 명칭과 속성을 정의합니다.
    - Standard Policy (표준 정책): 직급의 높낮이나 코드의 유효성 등 비즈니스 로직의 '판단 기준'이 되는 데이터를 관리합니다.
    - Cross-Domain Reference (교차 참조): 특정 도메인에 종속되지 않고, 모든 도메인(User, Approval, Shift 등)에서 가져다 쓰는 범용 데이터입니다.
- 외부 도메인과의 관계
    - MDM ↔ Approval: 결재 엔진은 MDM의 `Position.rankOrder`를 참조하여 승인자가 기안자보다 상급자인지 판단합니다.
    - MDM ↔ User: 사용자 도메인은 MDM의 `CommonCode`를 참조하여 자신의 '재직 상태(재직/휴직/퇴사)'를 표시합니다.
    - MDM ↔ Shift: 근무조 도메인은 MDM의 `LocationCode`를 참조하여 근무지가 어느 공장/라인인지 정의합니다.
- 유비쿼터스 언어 점검
    - Single Source of Truth (SSOT, 단일 진실 공급원): 동일한 데이터가 여러 곳에서 다르게 정의되지 않도록 MDM에서 관리하는 유일한 표준 데이터를 의미합니다.
    - Common Code (공통 코드): 시스템 전반에서 공유되는 상태값(예: 재직, 반려, 야간근무 등)을 그룹화하여 표준화한 코드 체계입니다.
    - Rank Order (서열 가중치): 직급 명칭(과장, 부장)과 별개로, 결재선 승인 권한이나 전결 규칙을 계산하기 위해 부여된 수치적 서열입니다.
    - Business Calendar (영업 달력): 일반 달력과 달리 공장 가동일, 전사 휴무일, 창립기념일 등 비즈니스 로직에 영향을 주는 날짜 정보가 포함된 달력입니다.
    - Data Dictionary (데이터 사전): 각 도메인에서 사용하는 용어의 정의, 타입, 제약 조건을 명시한 표준 가이드입니다.
    - Canonical Model (표준 모델): 서로 다른 시스템 간 데이터를 주고받을 때 기준으로 삼는 통합된 데이터 형식입니다.
- 액티비티 흐름
    
    
    | 구분 | ID | 액티비티 명 | 상세 내용 및 역할 |
    | --- | --- | --- | --- |
    | 정의 | V | 표준 코드 체계 관리 | 시스템 전반의 상태값(재직/휴직/완료 등) 및 구분자 규격화 |
    | 설계 | W | 직급 및 서열 정의 | 결재 및 권한 판단의 근거가 되는 직급 명칭과 서열 가중치(`rankOrder`) 확정 |
    | 운영 | X | 시스템 기준 일정 관리 | 공장 가동일, 공휴일 등 전사 공통 달력(Calendar) 관리 |

## Model

A. CommonCode (공통 코드)
시스템 전반의 상태값이나 구분자를 표준화합니다.

- `groupCode`: (예: USER_STATUS, SHIFT_TYPE, APPROVAL_RESULT)
- `code`: 실제 코드값 (예: 10, 20, 30)
- `codeName`: 코드 명칭 (예: 재직, 휴직, 퇴사)
- `useYn`: 현재 사용 여부
- `attribute1~3`: 코드별 추가 속성 (예: 색상 코드, 가중치 등)

B. Position & Rank (직급/직책 표준)
조직 내의 서열과 역할을 표준화합니다.

- `positionCode`: (예: POS_001, POS_002)
- `positionName`: (예: 사원, 대리, 과장, 부장)
- `rankOrder`: 서열 가중치 (예: 사원=1, 부장=10 / 승인 권한 및 전결 규칙의 핵심 지표)
- `dutyCode`: 직책 코드 (예: 팀장, 본부장, 파트장)

C. Business Calendar (영업/공장 달력)
시스템의 '시간적 기준'을 정의합니다. (Shift 도메인의 기초 데이터)

- `calDate`: 날짜 (PK)
- `dayType`: (예: 평일, 주말, 공휴일, 창립기념일)
- `isWorkDay`: 실제 가동일 여부

## Activity

V. 표준 코드 체계 관리 (Code Schema Management)

- 행위: 신규 비즈니스 로직에 필요한 코드 그룹을 생성하고 값을 정의합니다.
- 로직: 코드 간의 중복을 방지하고, 전 시스템이 동일한 코드값을 참조하도록 배포합니다.

W. 직급 및 서열 정의 (Rank & Authority Standard)

- 행위: 조직 내 모든 직급의 명칭과 그에 따른 서열(rankOrder)을 확정합니다.
- 비즈니스 영향: 여기서 설정된 서열에 따라 결재 도메인의 '상급자 자동 찾기'와 '전결 규칙'이 작동합니다.

X. 시스템 기준 일정 관리 (Standard Calendar Management)

- 행위: 연간 공휴일이나 공장 휴무일을 사전에 등록합니다.
- 비즈니스 영향: 근무조(Shift) 스케줄 자동 생성 시 이 달력을 참조하여 휴무일 배정을 자동화합니다.
